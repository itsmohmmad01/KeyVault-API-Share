How KeyVault Works
1
You add your API key (Admin)
Go to the Admin Panel → paste your key (OpenAI, Gemini, etc), give it a label, set the max call limit, and click Generate. The platform creates a unique share token.
2
Share the token (not the key)
Send the UUID token to your friend / classmate. They never see your actual API key — only the token.
3
Friend uses the token in their code
They call POST /api/token/<token>/use from their project. The server returns the API key and deducts one call from the quota.
4
You monitor and control
Track usage, reset counters, or revoke tokens instantly from the Admin Panel. Perfect for group projects, hackathons, or sharing with classmates.
💻
Use in Your Code (fetch example) 
// In your friend's project:
const res = await fetch('http://localhost:3000/api/token/YOUR_TOKEN/use', {
  method: 'POST'
});
const data = await res.json();
const apiKey = data.apiKey; // ← use this in your API call!
🔌
API Endpoints Reference
GET /api/token/:token/info Check token status & remaining calls
POST /api/token/:token/use Retrieve API key (uses 1 call)
POST /api/admin/create Create a new share token (admin)
GET /api/admin/list List all tokens (admin)
DELETE /api/admin/revoke/:token Revoke a token (admin)
